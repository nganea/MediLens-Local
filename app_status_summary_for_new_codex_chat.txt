Core app
--------
The app is called MediLens Local: Medicine Label Helper.

It is a local-first Gradio web app that helps read medicine labels from an uploaded or captured image. It extracts text from the image, matches the label against a local medicine database, and displays:

- Medicine match
- Medicine use
- Safety warning
- Source URL
- Extracted OCR text in Technical details

The app does not use cloud APIs.

Main user workflow
------------------
1. Upload or capture a medicine label image.
2. Optionally type the medicine name or label text.
3. Select the language.
4. Click Read medicine label.
5. The app checks local model server status.
6. MiniCPM-V and Tesseract work together to read the image.
7. The app matches the OCR/manual text against medicines.csv.
8. Tiny Aya optionally rewrites/translates the output.
9. The app displays medicine match, medicine use, safety warning, source URL, and OCR details.

OCR behavior
------------
MiniCPM-V 4.6 and Tesseract are enabled together by default.

Tesseract is useful for debugging and fallback text, but if MiniCPM-V cannot be reached or cannot read the image, the app should not trust Tesseract-only OCR for matching because it can produce incorrect medicine names.

If MiniCPM-V times out:
- The webpage stays open.
- The app tries to stop waiting for the model result.
- Medicine match shows N/A.
- Medicine use asks the user to type the medicine name or label text and search again.
- Safety warning shows N/A.
- The manual label field is highlighted.

When the language changes:

- The user must click Read medicine label again.

This avoids flickering and avoids showing output in the old language after a language change.

Translation behavior
--------------------
Tiny Aya is used to make the medicine use and safety warning easier to read and to translate them.

Desired explanation style:
- Like writing for a 14-15 year old.
- Maximum 3 short sentences.
- Non-repeating.
- No dosage instructions.
- Avoid unnecessary medicine form details such as tablets, creams, drops, or capsules unless clinically necessary.

Safety warning style:
- Avoid repeating generic and medicine-specific warnings.
- Use "patient information leaflet" rather than "patient leaflet".
- Do not tell the user that the medicine is safe for them personally.

Improve translation section
---------------------------
The app has an "Improve translation" section above Technical details.

It explains that Tiny Aya is useful for translation but not perfect.

Users can submit public glossary suggestions with:
- Language
- Bad phrase
- Preferred phrase

The public suggestions table loads when the app opens and updates immediately after a new suggestion is submitted. Suggestions are not automatically applied to the active glossary; they need human review first.

Known Romanian glossary example:
- Bad phrase: ochi lacrimogeni
- Preferred phrase: ochi inlacrimati

Safety position
---------------
The app is informational only.

It should not:
- Provide dosage instructions.
- Tell a user to take a medicine.
- Confirm that a medicine is safe for a specific person.
- Replace a pharmacist, doctor, or other qualified healthcare professional.

It should:
- Explain common medicine uses.
- Show source URL.
- Encourage checking with a pharmacist or doctor if unsure.
- Use N/A when the app cannot reliably identify or process the medicine.

Hands-free conversation flow
----------------------------
1. Loads + calibrates the mic (stay quiet), then Reachy waggles its antennas
   and says "I am listening" - that is the go-ahead cue.
2. Wake cue = the name "Reachy" + a medicine word. Matching strips accents and
   matches word stems, so it handles the Romanian suffixed (enclitic) article
   ("medicamentul", "medicamentului", "medicamente") and German "Medikament",
   French "medicament", Italian "medicinale", Spanish "medicamento".
   Whisper mishearings of "Reachy" (Riki/Reiki/Richie...) are matched by prefix.
3. Reachy says "Okay", captures the label, does the thinking motion, identifies
   the medicine, and speaks the answer in the SAME language you asked in.
4. It keeps listening for the next question. A slow/timed-out identification no
   longer crashes - Reachy apologises and keeps listening.
5. Say "Reachy stop" (works in every language) to end: Reachy gives a goodbye,
   lowers its antennas, and exits.

Languages: English, Romanian, German, French, Italian, Spanish. With
--language auto, Whisper detects the language per question; non-English answers
are translated by Tiny Aya and spoken with that language's voice.

Microphone choice
-----------------
--mic laptop (default): listen through the laptop microphone.
--mic reachy: listen through Reachy Mini's own mic (ReSpeaker, 16 kHz stereo
  over WebRTC via mini.media.get_audio_sample()). Run test_reachy_mic.py first
  to confirm the robot actually streams its mic audio.
